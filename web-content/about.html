Project Overview

The Weldy Project demonstrates a distributed web infrastructure using AWS, Docker, and HAProxy. Three backend servers host this website, while a proxy server manages load balancing and uptime monitoring.

Technologies Used

  • Amazon Web Services (AWS)
  • Docker Containers
  • HAProxy Load Balancer
  • Ubuntu Linux

How the Load Balancer Works

The HAProxy proxy instance distributes requests among the three backend servers. This ensures:

  • High availability
  • Improved performance
  • Redundancy if a server goes down

Server Identification

This page (and the main page) shows which backend server is serving the request dynamically. For example:

Served by: ${SERVER_NAME}

This helps demonstrate that HAProxy is correctly balancing requests among the backend pool.

Project Purpose

The project teaches:

  • Infrastructure as Code with AWS CloudFormation
  • Containerization and deployment with Docker
  • Load balancing using HAProxy
  • Secure instance-to-instance networking in a VPC
